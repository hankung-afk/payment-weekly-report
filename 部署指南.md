# 固定入口页面 - 部署指南

## 🎯 方案说明

**问题**：Cloudflare Quick Tunnel 每次启动都会生成新的随机 URL，不方便记忆和分享。

**解决方案**：
1. 创建一个固定的入口页面（部署到 Cloudflare Pages）
2. 入口页面显示当前的临时 Tunnel URL
3. 每次启动 Tunnel 时自动更新入口页面的配置
4. 你只需记住固定的 Pages URL，每次访问它就能看到最新的 Tunnel 地址

**优点**：
- ✅ 固定的入口 URL（如 https://payment-weekly-report.pages.dev）
- ✅ 自动更新，无需手动操作
- ✅ 完全免费（Cloudflare Pages 免费）
- ✅ 无需购买域名

---

## 📦 方案1：部署到 Cloudflare Pages（推荐）

### 步骤1：创建 GitHub 仓库

1. **登录 GitHub**：https://github.com
2. **创建新仓库**：点击右上角 `+` → `New repository`
3. **仓库设置**：
   - Repository name: `payment-weekly-report`
   - Description: `支付周报网站访问入口`
   - 选择 `Public`
   - 不要勾选 "Add a README file"
4. **点击** `Create repository`

### 步骤2：上传文件到 GitHub

在终端执行以下命令：

```bash
cd "/Users/han.kung/Desktop/支付AI机器人/周报网站/入口页面"

# 初始化 Git
git init

# 添加所有文件
git add .

# 提交
git commit -m "Initial commit: 周报网站入口页面"

# 连接到 GitHub（替换成你的用户名）
git remote add origin https://github.com/你的用户名/payment-weekly-report.git

# 推送到 GitHub
git branch -M main
git push -u origin main
```

### 步骤3：部署到 Cloudflare Pages

1. **登录 Cloudflare**：https://dash.cloudflare.com
2. **进入 Pages**：左侧菜单 → `Workers & Pages` → `Create application` → `Pages`
3. **连接 GitHub**：
   - 点击 `Connect to Git`
   - 选择 `GitHub`
   - 授权 Cloudflare 访问你的 GitHub
4. **选择仓库**：
   - 选择刚才创建的 `payment-weekly-report`
   - 点击 `Begin setup`
5. **配置构建设置**：
   - Project name: `payment-weekly-report`（或自定义）
   - Production branch: `main`
   - Build settings: 保持默认（不需要构建）
   - Root directory: `/`
6. **点击** `Save and Deploy`

### 步骤4：获取固定 URL

部署完成后，你会得到一个固定的 URL：

```
https://payment-weekly-report.pages.dev
```

或者自定义为：

```
https://你的项目名.pages.dev
```

🎉 **完成！** 这就是你的固定入口地址！

## 🔄 使用流程

### 启动服务

双击运行：**`启动公网访问_固定入口.command`**

这个脚本会：
1. 启动本地 HTTP 服务器
2. 启动 Cloudflare Tunnel
3. **自动提取 Tunnel URL**
4. **自动更新 `config.json`**

### 自动同步到 Pages

每次启动后，需要把更新后的 `config.json` 推送到 GitHub：

```bash
cd "/Users/han.kung/Desktop/支付AI机器人/周报网站/入口页面"

git add config.json
git commit -m "Update tunnel URL"
git push
```

Cloudflare Pages 会自动检测到更新并重新部署（约1分钟）。

### 访问

访问你的固定 URL：`https://payment-weekly-report.pages.dev`

页面会：
1. 显示当前的 Tunnel URL
2. 提供 "立即访问" 按钮
3. 提供 "复制地址" 功能
4. 显示更新时间

## 📦 方案2：使用 GitHub Pages（备选）

如果不想用 Cloudflare Pages，也可以用 GitHub Pages：

### 步骤：

1. 在 GitHub 仓库 → `Settings` → `Pages`
2. Source: `Deploy from a branch`
3. Branch: `main` → `/` (root)
4. 点击 `Save`

你的固定 URL 会是：
```
https://你的用户名.github.io/payment-weekly-report/
```

**区别**：
- GitHub Pages: 5-10分钟部署
- Cloudflare Pages: 1分钟部署（更快）

## ⚠️ 注意事项

1. **GitHub 仓库建议设为 Public**
   - Cloudflare Pages 免费版只支持 Public 仓库
   - 入口页面不包含敏感信息，可以公开

2. **config.json 更新频率**
   - 只在启动 Tunnel 时更新
   - 不包含敏感数据（只有 URL 和时间戳）

3. **Tunnel 仍然是临时的**
   - 固定的只是入口 URL
   - 实际的 Tunnel URL 每次启动都不同
   - 关闭服务后 Tunnel URL 失效

4. **延迟时间**
   - 启动 Tunnel → 约5秒
   - 推送到 GitHub → 即时
   - Cloudflare Pages 部署 → 约1分钟
   - 总延迟：约1分钟
