# 周报网站 - 固定访问入口

这是周报网站的固定访问入口页面，解决 Cloudflare Quick Tunnel 每次地址都变化的问题。

## 📁 文件说明

```
入口页面/
├── index.html              # 主入口页面（自动跳转）
├── tunnel-url.html         # 显示当前 Tunnel URL
├── config.json            # 存储当前的 Tunnel URL（自动生成）
├── 部署指南.md             # 详细的部署步骤
├── 自动更新到云端.command   # 一键推送更新到 GitHub
└── README.md              # 本文件
```

## 🚀 快速开始

### 第一次使用

1. **阅读部署指南**：打开 `部署指南.md`
2. **创建 GitHub 仓库**：按指南步骤操作
3. **部署到 Cloudflare Pages**：免费，无需域名
4. **获得固定 URL**：如 `https://payment-weekly-report.pages.dev`

### 日常使用

1. **启动服务**：
   ```bash
   双击运行: 启动公网访问_固定入口.command
   ```

2. **推送更新**：
   ```bash
   双击运行: 自动更新到云端.command
   ```

3. **访问网站**：
   ```
   访问你的固定 URL (Cloudflare Pages)
   → 看到当前的 Tunnel 地址
   → 点击"立即访问"
   → 进入周报网站
   ```

## 💡 工作原理

```
┌─────────────────┐
│  固定 URL        │  ← 你和同事记住这个地址
│  (Pages 部署)    │
└────────┬────────┘
         │
         ↓ 读取 config.json
         │
┌────────┴────────┐
│ 当前 Tunnel URL │  ← 每次启动自动更新
│ (临时地址)       │
└────────┬────────┘
         │
         ↓ 访问
         │
┌────────┴────────┐
│  周报网站        │
│  (本地服务)      │
└─────────────────┘
```

## ⚡ 优势

- ✅ **固定入口**：URL 永远不变
- ✅ **自动更新**：启动时自动更新配置
- ✅ **完全免费**：Cloudflare Pages 免费
- ✅ **无需域名**：使用 `.pages.dev` 子域名
- ✅ **快速部署**：约1分钟完成更新

## 🔧 故障排查

### 页面显示"暂无可用地址"

**原因**：config.json 未更新或服务未启动

**解决**：
1. 确认已运行 `启动公网访问_固定入口.command`
2. 运行 `自动更新到云端.command`
3. 等待1分钟后刷新页面

### 推送失败

**原因**：Git 配置或网络问题

**解决**：
1. 检查 GitHub 登录状态
2. 确认有推送权限
3. 检查网络连接
4. 手动执行：
   ```bash
   cd 入口页面
   git push
   ```

### 部署失败

**原因**：Cloudflare Pages 配置问题

**解决**：
1. 检查仓库是否为 Public
2. 确认 Build settings 为空（静态站点）
3. 重新触发部署
